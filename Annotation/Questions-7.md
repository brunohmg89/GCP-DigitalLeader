3. You are working on implementing a solution that requires you to use a scalable, no-maintenance and serverless document database. Which of the below Google Cloud products would you use?<br>
    _Você está trabalhando na implementação de uma solução que exige o uso de um banco de dados de documentos escalável, sem manutenção e sem servidor. Quais dos produtos do Google Cloud abaixo você usaria?_

    :white_check_mark: Firestore

    :black_large_square: Cloud Spanner

    :black_large_square: Memorystore

    :black_large_square: Cloud Bigtable

    > Option A is correct. Firestore is a Google product that is scalable, requires no maintenance, and it is a serverless document database.<br>
    
    > _A opção A está correta. Firestore é um produto do Google escalonável, não requer manutenção e é um banco de dados de documentos sem servidor._

    > Option B is incorrect. Cloud Spanner is incorrect because it is a relational database service, not a document database.<br>
    
    > _A opção B está incorreta. O Cloud Spanner está incorreto porque é um serviço de banco de dados relacional, não um banco de dados de documentos._

    > Option C is incorrect. Memorystore, also known as Memorystore for Redis, is a fully managed Google’s in-memory data store service.<br>
    
    > _A opção C está incorreta. Memorystore, também conhecido como Memorystore for Redis, é um serviço de armazenamento de dados na memória do Google totalmente gerenciado._
    
    > Option D is incorrect. Cloud Bigtable is a big data database service, not a document database.
    
    > _A opção D está incorreta. O Cloud Bigtable é um serviço de banco de dados de big data, não um banco de dados de documentos._

6. Uma organização quer analisar os dados coletados de veículos de aluguel ou de uma frota. Integre os sensores para enviar os dados de local a cada 30 segundos. Ela quer ingerir esses dados. Qual produto ou serviço do Google ela deve usar?

    :black_large_square: Dataflow

    :white_check_mark: Pub/Sub

    :black_large_square: Cloud SQL

    :black_large_square: Cloud Storage

    > A opção A está incorreta porque o Dataflow é usado para transformar e processar dados depois de recebidos, e não para ingeri-los.

    > A opção B está correta porque o Pub/Sub é um serviço de mensagens que permite receber dados de streams de dispositivos, como sensores, no início de um pipeline.

    > A opção C está incorreta porque o Cloud SQL é um banco de dados relacional e não é adequado para grandes quantidades de dados do sensor.

    > A opção D está incorreta porque, embora o Cloud Storage possa ser usado para armazenar dados, para ingestão confiável é preciso ter um serviço de mensagens.

10. Como a privacidade é definida no contexto da tecnologia de nuvem?

    :black_large_square: Procedimentos para autenticar a identidade do usuário

    :white_check_mark: Restrições de compartilhamento e acesso aos dados

    :black_large_square: Conformidade com normas regulamentares

    :black_large_square: Suscetibilidade a violação de dados e ataques cibernéticos

    > A opção A está incorreta porque a privacidade não é a autenticação de uma identidade do usuário, e sim a restrição de acesso aos dados.

    > A opção B está correta porque, no contexto de tecnologia de nuvem, a privacidade significa que há restrições no acesso aos dados e compartilhamento.

    > A opção C está incorreta porque a conformidade é quando uma organização precisa cumprir as leis e regulamentações locais para dados ou operações, e privacidade é a restrição de acesso aos dados.

    > A opção D está incorreta porque a suscetibilidade a violações de dados e ataques cibernéticos mede a probabilidade de o evento ocorrer, e a privacidade é a restrição de acesso aos dados.
